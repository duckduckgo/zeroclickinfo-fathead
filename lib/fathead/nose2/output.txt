installation	A							http://nose2.readthedocs.io/en/latest/getting_started.html			<p>The recommended way to install nose2 is with <a  href="http://pypi.python.org/pypi/pip/1.0.2">pip</a></p><p>You can also install from source by downloading the source distribution from <a  href="http://pypi.python.org/pypi">pypi</a>, un-taring it, and running <code <span python</span> <span setup.py</span> <span install</span></code> in the source directory. Note that if you install this way, and do not have distribute or setuptools installed, you must install nose2’s dependencies manually.</p><p>For Python 2.7, Python 3.2 and pypy, nose2 requires <a  href="http://pypi.python.org/pypi/six/1.1.0">six</a> version 1.1. For Python 2.6, nose2 also requires <a  href="http://pypi.python.org/pypi/argparse/1.2.1">argparse</a> version 1.2.1 and <a  href="http://pypi.python.org/pypi/unittest2/0.5.1">unittest2</a> version 0.5.1. When installing with pip, distribute or setuptools, these dependencies will be installed automatically.</p><p>You can install the development version of nose2 from github with <a  href="http://pypi.python.org/pypi/pip/1.0.2">pip</a>:</p><p>You can also download a package from github, or clone the source and install from there with <code <span python</span> <span setup.py</span> <span install</span></code>.</p>	http://nose2.readthedocs.io/en/latest/getting_started.html#installation

running tests	A							http://nose2.readthedocs.io/en/latest/getting_started.html			<p>To run tests in a project, use the <code <span nose2</span></code> script that is installed with nose2:</p><p>This will find and run tests in all packages in the current working directory, and any sub-directories of the current working directory whose names start with ‘test’.</p><p>To find tests, nose2 looks for modules whose names start with ‘test’. In those modules, nose2 will load tests from all <a  href="http://docs.python.org/library/unittest.html#unittest.TestCase" title="(in Python v2.7)"><code  py-class docutils literal"><span unittest.TestCase</span></code></a> subclasses, as well as functions whose names start with ‘test’.</p><p>The <code <span nose2</span></code> script supports a number of command-line options, as well as extensive configuration via config files. For more information see <a  href="usage.html"><em>Using nose2</em></a> and <a  href="configuration.html"><em>Configuring nose2</em></a>.</p>	http://nose2.readthedocs.io/en/latest/getting_started.html#running-tests
