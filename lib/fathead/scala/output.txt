Application	A										<section class="prog_container"><p>The <code>Application</code> trait can be used to quickly turn objects\n into executable programs, but is not recommended.\n Here is an example:</p><pre><code>object Main extends Application {\nConsole.println("Hello World!")\n}</code></pre><p> Here, object <code>Main</code> inherits the <code>main</code> method of <code>Application</code>.\n The body of the <code>Main</code> object defines the main program. This technique\n does not work if the main program depends on command-line arguments\n (which are not accessible with the technique presented here).</p><p> It is possible to time the execution of objects that inherit from class\n <code>Application</code> by setting the global <code>scala.time</code>\n property. Here is an example for benchmarking object <code>Main</code>:</p><pre><code>java -Dscala.time Main</code></pre><p> In practice the <code>Application</code> trait has a number of serious pitfalls:</p><ul class='prog_ul'><li>Threaded code that references the object will block until static\n   initialization is complete.  However, because the entire execution\n   of an <pre><code>object</code></pre> extending <pre><code>Application</code></pre> takes place during\n   static initialization, concurrent code will always deadlock if\n   it must synchronize with the enclosing object.</li><li>As described above, there is no way to obtain the\n   command-line arguments because all code in body of an <pre><code>object</code></pre>\n   extending <pre><code>Application</code></pre> is run as part of the static initialization\n   which occurs before <pre><code>Application</code></pre>'s <pre><code>main</code></pre> method\n   even begins execution.</li><li>Static initializers are run only once during program execution, and\n   JVM authors usually assume their execution to be relatively short.\n   Therefore, certain JVM configurations may become confused, or simply\n   fail to optimize or JIT the code in the body of an <pre><code>object</code></pre> extending\n   <pre><code>Application</code></pre>.  This can lead to a significant performance degradation.</li></ul><p> It is recommended to use the scala.App trait instead.</p><pre><code>object Main {\ndef main(args: Array[String]) {\n  //..\n}\n}</code></pre></section>	https://www.scala-lang.org/api/current/scala/Application.html
